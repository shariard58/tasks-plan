## 🗺️ কিভাবে এই Journey শুরু করবে

### Step 1: Environment Setup (Day 1)

```
- VS Code install করো + extensions: ESLint, Prettier, GitLens, Thunder Client
- Node.js LTS install করো
- Git install করো + GitHub account বানাও
- Terminal/Command Line comfortable হও
- pnpm install করো (npm এর চেয়ে faster)
```

### Step 2: Daily Routine বানাও

```
সকাল (2-3 ঘন্টা):  নতুন concept শেখো + task শুরু করো
বিকাল (2-3 ঘন্টা):  task complete করো + debug করো
রাত (1-2 ঘন্টা):    code review করো + পরের দিনের plan করো

Weekend: আগের সপ্তাহের tasks revisit + refactor
```

### Step 3: প্রতিটা Task এ যা করবে

```
1. Requirements ভালো করে পড়ো
2. চিন্তা করো কিভাবে approach করবে (10-15 min)
3. Pseudocode/rough plan লেখো
4. Code করো
5. Test করো (manually + edge cases)
6. Refactor করো
7. Git commit করো proper message দিয়ে
8. README.md লেখো (কী শিখলে, কী কঠিন ছিল)
```

## 🔥 Golden Rules (এগুলো মেনে চলো)

### Rule 1: Copy-Paste করবে না

- Documentation পড়ো, বুঝো, তারপর নিজে লেখো
- Stack Overflow থেকে concept নাও, code না

### Rule 2: প্রতিটা Error তোমার Teacher

- Error আসলে frustrated হয়ো না
- Error message পড়ো, Google করো, বুঝো কেন আসলো
- Error fix করার process মনে রাখো

### Rule 3: Git হলো তোমার Best Friend

- প্রতিটা meaningful change এ commit করো
- Branch ব্যবহার করো features এর জন্য
- Commit messages meaningful লেখো

### Rule 4: TypeScript ব্যবহার করো Phase 02 থেকে

- JavaScript জানলে TypeScript শেখা সহজ
- Real-world এ TypeScript ছাড়া job নেই
- Phase 02 থেকে সব task TypeScript এ করো

### Rule 5: Mobile-First Design Always

- প্রতিটা UI task এ mobile-first approach
- Chrome DevTools responsive mode ব্যবহার করো
- Real device এ test করো

### Rule 6: Documentation পড়ার অভ্যাস করো

- Official docs সবসময় first priority
- React docs, Next.js docs, MDN — এগুলো তোমার Bible

### Rule 7: Code Review নিজে নিজে করো

- একদিন পর নিজের code আবার পড়ো
- কি improve করা যায় চিন্তা করো
- Refactor করো

## 💡 যখন Stuck হবে তখন কী করবে

```
1. Error message ভালো করে পড়ো
2. Console.log দিয়ে debug করো
3. Google এ exact error message search করো
4. Stack Overflow check করো
5. Official documentation পড়ো
6. 30 min চেষ্টার পর break নাও (15 min)
7. Fresh mind এ আবার try করো
8. তবুও না পারলে concept আবার পড়ো
9. Similar examples দেখো (copy না, বুঝো)
10. Solve করার পর নোট লেখো — কী শিখলে
```
