## 🔥 Golden Rules

1. **"Express শেখার আগে raw HTTP server বানাও"** — Framework magic বোঝার আগে fundamentals বোঝো
2. **"Query লেখার আগে database internals বোঝো"** — Index কেন কাজ করে জানলে slow query কখনো লিখবে না
3. **"Security পরে শিখবো বলো না"** — Phase 08 skip করলে production এ hack হবে
4. **"System Design = Interview Pass"** — FAANG/top companies এ system design না জানলে entry নেই
5. **"Docker ছাড়া modern backend নেই"** — Deployment না জানলে code লিখে লাভ নেই
6. **"প্রতিটা task নিজে করো, copy-paste = career destroy"**
7. **"AI কে tool হিসেবে use করো, brain হিসেবে না"**

## ✅ Task Process (প্রতিটা Task এভাবে করো)

```
1. Task পড়ো — কী করতে হবে clearly বোঝো
2. নিজে try করো — minimum 30 মিনিট struggling
3. Research করো — docs, articles, videos
4. Implement করো — code লেখো, test করো
5. Edge cases handle করো — error handling, validation
6. Document করো — কী শিখলে, কী mistake করেছিলে
7. Next task এ যাও
```

## ❌ Common Mistakes (এগুলো কোরো না)

| Mistake                                | Reality                                               |
| -------------------------------------- | ----------------------------------------------------- |
| "শুধু Express শিখলেই backend হয়"      | Express ছাড়াও raw Node.js, Fastify, Koa জানতে হবে    |
| "MongoDB সব জায়গায় use করবো"         | SQL + NoSQL দুটোই জানতে হবে, use case বুঝতে হবে       |
| "Security পরে শিখবো"                   | হ্যাক হলে পরে বলে লাভ নেই                             |
| "Docker skip করবো"                     | Modern backend = containerized backend                |
| "System Design শুধু interview এর জন্য" | Real production system design = career differentiator |
| "Testing boring"                       | Testing ছাড়া production code = bomb                  |
| "Linux জানার দরকার নেই"                | Backend server = Linux server (99%)                   |
