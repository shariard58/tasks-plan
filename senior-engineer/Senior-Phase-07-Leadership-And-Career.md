## Section A: Technical Leadership (Tasks 216-230)

### Task 216 — Tech Lead Responsibilities 🟡

- Code quality ownership, Architecture decisions, Unblocking team
- Balance: coding (40%) + reviewing (30%) + planning (20%) + mentoring (10%)
- **Exercise:** Document: "What I'll do differently as a Tech Lead"

### Task 217 — Architecture Decision Making 🟡

- Evaluate tradeoffs: performance vs simplicity vs cost vs time
- Consider: team skill, timeline, maintenance, scaling needs
- **Exercise:** 5 টা architecture decision evaluate করো (pros/cons/tradeoffs)

### Task 218 — Technical Roadmap Planning 🟡

- Quarter-level technical planning
- Balance: features vs tech debt vs infrastructure vs learning
- **Exercise:** 3-month technical roadmap for a project

### Task 219 — Mentoring Junior Developers 🟡

- Pair programming, Code review teaching, Assignment design
- Don't give answers — guide them to find answers
- **Exercise:** Mentoring plan for a junior developer (3-month program)

### Task 220 — Agile/Scrum Mastery 🟡

- Sprint planning, Daily standup, Sprint review, Retrospective
- Story writing, Acceptance criteria, Definition of Done
- **Exercise:** Run a mock sprint: planning → daily → review → retro

### Task 221 — Sprint Planning & Estimation 🟡

- User stories, Story points, Velocity tracking
- Breaking down large tasks into estimatable pieces
- **Exercise:** 20 টা user story লেখো acceptance criteria সহ

### Task 222 — Technical Debt Management 🟡

- Identify, categorize, prioritize, schedule tech debt work
- "20% rule": spend 20% of sprint on tech debt
- **Exercise:** Tech debt audit: categorize ও create payoff plan

### Task 223 — Incident Management 🟡

- Severity levels, On-call rotation, Incident commander role
- Runbook, Escalation path, Post-mortem
- **Exercise:** Incident response plan document

### Task 224 — On-Call Best Practices 🟡

- Monitoring ও alerting setup, Runbooks, Escalation
- Work-life balance on-call, Alert fatigue prevention
- **Exercise:** On-call handbook for team

### Task 225 — Cross-Team Collaboration 🟡

- Working with Product, Design, QA, DevOps teams
- API contracts, Shared documentation, Communication protocols
- **Exercise:** Cross-team collaboration guide

### Task 226 — Project Estimation ও Planning 🟡

- Bottom-up estimation, Buffer (add 30-50%), Risk assessment
- **Exercise:** Full project estimate: features, timeline, resources, risks

### Task 227 — Code Standards ও Guidelines 🟡

- Team coding standards document
- Naming, Structure, Error handling, Testing, Git conventions
- **Exercise:** Complete coding standards guide for your team/project

### Task 228 — Knowledge Sharing 🟢

- Tech talks, Lunch & Learn, Documentation, Wiki
- **Exercise:** 3 tech talks prepare করো team এর জন্য

### Task 229 — Performance Reviews (Giving ও Receiving) 🟡

- Constructive feedback, Growth areas, Strengths
- **Exercise:** Self-assessment ও peer review practice

### Task 230 — Building Engineering Culture 🟡

- Innovation time, Hackathons, Learning culture, Psychological safety
- **Exercise:** Engineering culture proposal for your team

## Section B: Interview Mastery (Tasks 231-240)

### Task 231 — Resume/CV Optimization 🟡

- One page, Impact-focused (numbers!), Relevant keywords
- ```
  ❌ "Worked on backend using Node.js"
  ✅ "Built REST API handling 50K+ requests/sec using Node.js, PostgreSQL,
     Redis — reduced response time by 40%"
  ```
- **Exercise:** Resume rewrite: every bullet point with measurable impact

### Task 232 — Portfolio Building 🟡

- GitHub profile: pinned repos, green contribution graph, README profile
- Personal website: projects, blog, about, contact
- **Exercise:** GitHub profile + Personal portfolio site তৈরি করো

### Task 233 — Behavioral Interview (STAR Method) 🟡

- **S**ituation → **T**ask → **A**ction → **R**esult
- Common questions:
  - Tell me about a challenging project
  - Describe a conflict with a teammate
  - Tell me about a time you failed
  - How do you handle tight deadlines
  - Describe a time you mentored someone
- **Exercise:** 20 টা STAR stories prepare করো

### Task 234 — System Design Interview 🔴

- Framework: Requirements → Estimation → High-Level Design → Deep Dive → Tradeoffs
- Practice problems: URL shortener, Chat system, News feed, File storage
- **Exercise:** 10 টা system design problem practice করো (45 min each)

### Task 235 — Live Coding Interview Tips 🟡

- Think aloud, Clarify requirements, Write tests first, Optimize later
- Handle: stuck moments, hints, time pressure
- **Exercise:** Mock coding interviews (timer সেট করে practice)

### Task 236 — Company Research 🟢

- Before interview: company tech stack, engineering blog, culture, products
- **Exercise:** 5 টা target company deeply research করো

### Task 237 — Salary Negotiation 🟡

- Know your market value (Glassdoor, levels.fyi)
- Negotiate total compensation: base + bonus + equity + benefits
- Never accept first offer, Always negotiate politely
- **Exercise:** Mock salary negotiation practice

### Task 238 — Interview Questions to Ask 🟢

- "What does the engineering team look like?"
- "How do you handle technical debt?"
- "What's the CI/CD pipeline like?"
- "How do you do code reviews?"
- **Exercise:** 15 insightful questions prepare for interviewers

### Task 239 — Remote Interview Skills 🟢

- Setup: good internet, quiet room, camera, lighting
- Screen sharing, Collaborative coding (CoderPad, HackerRank)
- **Exercise:** Mock remote interview setup ও practice

### Task 240 — Interview Post-Mortem 🟢

- After each interview: what went well, what didn't, what to improve
- **Exercise:** Interview tracking spreadsheet তৈরি করো

## Section C: Career Growth (Tasks 241-250)

### Task 241 — Career Path Awareness 🟢

- IC track: Junior → Mid → Senior → Staff → Principal → Distinguished
- Management track: Tech Lead → Engineering Manager → Director → VP → CTO
- **Exercise:** Career path plan: where are you now, where do you want to be in 5 years

### Task 242 — Personal Branding 🟡

- "What are you known for?" — Pick 2-3 specialties
- Consistent presence: blog, Twitter/X, LinkedIn, GitHub
- **Exercise:** Personal brand statement + content calendar (1 post/week)

### Task 243 — LinkedIn Optimization 🟢

- Headline, About, Experience (impact-focused), Skills, Recommendations
- Post regularly: learnings, projects, opinions
- **Exercise:** LinkedIn profile overhaul ও weekly posting plan

### Task 244 — Networking in Tech 🟡

- Attend meetups, Conferences (local ও online)
- Twitter/X tech community, Discord servers
- Give talks, Contribute to discussions
- **Exercise:** Join 3 communities, attend 2 meetups, make 10 connections

### Task 245 — Conference Talks 🔴

- CFP (Call for Papers) submission, Talk preparation
- Start small: local meetup → online conference → in-person conference
- **Exercise:** Submit 1 CFP ও prepare a 30-min talk

### Task 246 — Side Projects Strategy 🟡

- Build things that solve real problems
- Ship fast, iterate, get users
- **Exercise:** Launch 1 side project (SaaS, Tool, or Library)

### Task 247 — Freelancing Basics 🟡

- Upwork/Fiverr/Toptal profiles, Pricing, Client communication
- Contracts, invoicing, time tracking
- **Exercise:** Freelancing profile setup ও first project

### Task 248 — Open Source Reputation 🟡

- Maintainer of popular repos = instant credibility
- Regular contributions, helpful issues/PRs
- **Exercise:** Become regular contributor to 2 projects

### Task 249 — Continuous Learning Strategy 🟢

- Learning plan: 1 hour/day, 1 new thing/week, 1 project/month
- Sources: newsletters, podcasts, blogs, courses
- **Exercise:** 1-year learning plan with quarterly goals

### Task 250 — Build: Complete Professional Profile ⚫

- **Phase 7 Final Project:**
  - GitHub: Optimized profile, 10+ quality repos, contribution graph
  - Portfolio: Personal website (projects, blog, resume, contact)
  - LinkedIn: Optimized, recommendations, posts
  - Blog: 10+ published articles
  - Resume: Impact-focused, ATS-friendly, tailored versions
  - Open Source: Active contributor/maintainer
  - Network: 50+ meaningful connections in tech
  - Interview Prep: 20 STAR stories, 10 system design solutions
  - **তুমি এখন job market এ একটা brand — companies তোমাকে খুঁজবে**
